# G-OpenSteamTool v0.2.3

v0.2.3 是一次面向 Windows 桌面使用体验的正式版本更新，重点优化管理器 UI 工程、窗口尺寸一致性、控件动效和在线更新体验。

## 主要更新

- 将应用版本更新为 `0.2.3`，同步前端显示版本、Tauri 配置、Cargo 元数据和 npm lockfile。
- 重构主界面为更清晰的桌面工具布局：左侧图标导航 + 右侧工作区，移除不必要的上下文侧栏。
- 统一 Tauri EXE 窗口与 UI 画布尺寸为 `1380 x 860`，修复 EXE 中外圈留白、顶部区域压缩和画布错位问题。
- 停用最大化入口：标题栏仅保留最小化和关闭，双击标题栏不再触发最大化。
- 优化左侧导航：
  - 导航按钮固定为 `44px x 44px` 正方形。
  - 图标、激活背景和左侧指示条居中对齐。
  - 修复按钮排列不均衡、选中态偏移的问题。
- 重做开关控件为机械拨杆风格，统一 Lua、设置和更新页面中的开关视觉与交互。
- 新增自定义下拉组件，修复原生下拉菜单在日志页和 TOML 设置页中的错位问题。
- 优化按钮、列表、表格、输入框、滚动条、状态控制台等核心交互动效。
- 状态控制台新增通知切换动效：新状态从下方出现，旧状态向上淡出；长文本仍支持横向滚动。
- Auto-Fetcher 在概览页标记为“功能暂未实现”，避免误导用户依赖尚未完成的自动获取能力。
- 保留关于页 Logo/插图原始颜色，不再对其套用绿色滤镜。
- 新增 `npm run build:tauri-release`，用于本地生成 `src-tauri/target/release` 下的正式构建产物。

## 修复

- 修复 EXE 窗口和 UI 尺寸不匹配导致的白边和裁切问题。
- 修复左侧导航按钮不是实际正方形、图标视觉偏移的问题。
- 修复多个开关滑块与轨道未垂直居中的问题。
- 修复日志页滚动条主题残留紫蓝色的问题，同时保留日志内容原有颜色。
- 修复 Lua 表格、DLL 校验区域和 TOML 表单控件的深色主题不一致问题。
- 修复概览页部分卡片内容过紧但留白失衡的问题。
- 修复 Auto-Fetcher 占位卡片渐变过重的问题，改为更克制的纯深色占位。

## 验证
